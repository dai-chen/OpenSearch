# Codebase Patterns
(Patterns discovered during implementation will be consolidated here)

## Known Patterns from Design Phase
- OpenSearch modules live in `modules/<name>/` with `build.gradle` using `opensearchplugin {}` block
- Modules are registered in root `settings.gradle` via `addSubProjects` or similar
- REST handlers extend `BaseRestHandler`, override `routes()` and `prepareRequest()`
- Plugin classes implement `ActionPlugin` and register handlers via `getRestHandlers()`
- Reference modules: `modules/reindex/`, `modules/lang-painless/` for build.gradle patterns
- The unified-query-api Maven coordinates: `org.opensearch.query:unified-query-api` (check `sandbox/plugins/analytics-engine/build.gradle` for exact repo URL and version)
- The IT pattern for unified query: `UnifiedQueryPlanner.plan()` → `UnifiedQueryCompiler.compile()` → `PreparedStatement.executeQuery()` → `ResultSet`
- Schema is built as a lazy `AbstractSchema` that creates `OpenSearchIndex(osClient, settings, indexName)` on demand
- Use `OpenSearchNodeClient` (not REST client) since the module runs in-process
- OpenSearch jar hell checker will reject duplicate classes — resolve via exclusions and resolutionStrategy.force
- Integration tests go in `src/internalClusterTest/java/`, class names end with "IT", extend `OpenSearchIntegTestCase`

## Discovered Patterns (US-001)
- `settings.gradle` auto-discovers modules — just create `build.gradle` in `modules/<name>/`, no settings.gradle edit needed

## Discovered Patterns (US-002)
- `unified-query-opensearch` jar (org.opensearch.query:unified-query-opensearch) contains the full opensearch module: OpenSearchIndex, OpenSearchNodeClient, CalciteLogicalIndexScan, OpenSearchDataType, etc.
- No need to copy source files from the sql repo — just add the jar as a dependency with appropriate exclusions
- Constructor pattern: `new OpenSearchIndex(new OpenSearchNodeClient(nodeClient), settings, indexName)`
- Schema pattern: lazy `AbstractSchema` with `getTableMap()` returning `OpenSearchIndex` on demand, passed to `UnifiedQueryContext.builder().catalog(name, schema)`
- Guava forbidden on compile classpaths — use custom `calciteCompile` configuration, add to sourceSets manually
- Per-dependency exclusions for server-provided deps (jackson, log4j, slf4j, etc.) — NOT global `configurations.all` excludes (breaks test framework)
- `forbiddenApisMain` needs all referenced classes on compile classpath — exclude server deps from runtime only, not compile
- `thirdPartyAudit.ignoreMissingClasses()` for server-provided classes, `ignoreViolations()` for guava Unsafe
- `dependencyLicenses { mapping from: /pattern/, to: 'name' }` groups jars under shared LICENSE/NOTICE
- `./gradlew :modules:query-languages:updateShas` auto-generates `.jar.sha1` files
- `testingConventions` requires at least one test in both `test` and `internalClusterTest` source sets
- All public classes/methods need javadoc with `@param` tags (enforced by `missingJavadoc`)
- Package needs `package-info.java` with javadoc

## Discovered Patterns (US-003)
- Action names MUST use valid prefixes: `indices:data/read/`, `indices:data/write/`, `cluster:admin/`, `cluster:monitor/`, `indices:admin/`, `indices:monitor/`, `internal:`, `indices:internal/`, `cluster:internal/`, `views:` — anything else (e.g. `cluster:data/`) is rejected by the test framework
- Transport action is reusable for SQL and PPL — differentiated by `QueryType` enum in the request
- `HardcodedSettings` extends `org.opensearch.sql.common.setting.Settings` for PoC defaults
- Constructor javadoc required even for no-arg constructors on public classes

## Discovered Patterns (US-005)
- Setting key strings must match `Settings.Key` enum's `keyValue` field (e.g., `plugins.calcite.enabled` not `calcite.engine.enabled`)
- `QUERY_MEMORY_LIMIT` expects `ByteSizeValue`, `SQL_CURSOR_KEEP_ALIVE` expects `TimeValue`
- `jackson-databind` is NOT provided by OpenSearch server — must be bundled for unified-query-opensearch
- The `sql_background_io` thread pool must be registered via `getExecutorBuilders()` with `ScalingExecutorBuilder`
- `CALCITE_PUSHDOWN_ROWCOUNT_ESTIMATION_FACTOR` (Double) must be in HardcodedSettings
- Integration tests can use transport client directly: `client().execute(QueryLanguageAction.INSTANCE, request).actionGet()`

## Discovered Patterns (US-006)
- `UnifiedQueryPlanner.buildQueryParser()` only supports PPL — for SQL, use Calcite's native `Frameworks.getPlanner(config)` → parse → validate → rel
- Calcite's default SQL parser uppercases unquoted identifiers — use `Lex.JAVA` for case-insensitive matching
- `Frameworks.newConfigBuilder(baseConfig)` can override parser config from existing FrameworkConfig
- `UnifiedQueryCompiler.compile()` accepts any RelNode regardless of source (PPL or Calcite SQL planner)

## GUARDRAILS — DO NOT VIOLATE
- ONLY modify files inside `modules/query-languages/` (exception: `settings.gradle` for module registration in US-001 only)
- Do NOT modify any existing module, plugin, or server code
- Do NOT modify `sandbox/plugins/analytics-engine/` — it is unrelated to this work
- Do NOT add dependencies to the root build.gradle or any other module's build.gradle
- Do NOT delete or modify any existing tests
- Keep the implementation minimal — this is a PoC, not production code
- Use hardcoded default settings, not configurable plugin settings
- No explain support, no cursor/pagination, no async query
- If jar conflicts arise, resolve via exclusions/force in THIS module's build.gradle only
